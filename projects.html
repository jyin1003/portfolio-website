FINANCIAL PORTFOLIO ANALYSIS
Project Link
Tech Stack
Python:
Web Scraping
Data Analysis
Sentiment Analysis
Financial Optimisation
Overview

Provide technical analysis to support a macro-driven hypothesis and portfolio strategy.

Model the projection of assets using machine-learning, quantify geopolitical sentiments and optimise portfolio allocations to achieve maximum returns per unit of risk.

Process

  1. Conducted sentiment analysis by building a web-scraping tool that collated ~100 related news articles. Implement a counter to track word frequencies over time.
  2. Model asset class projections by collecting price data and then splitting the data 80:20 to train and backtest the machine learning model respectively.
  3. Optimised portfolio allocations using the Hierachial Risk Parity technique to derive weightings that would inform fund allocation decisions.

Challenges

  1. Not all images had price data that could be pulled from Yahoo Finance. Instead, I was able to find online CSV files and then concatenate them into a single dataframe with an index of DateTime values.
  2. The linear prediction didn't extend to the portfolio's intended 3-month horizon so I used a while loop to continuously iterate through the latest predicted price, apply the trained linear regression model, and then append to the dataframe.
  3. Initially I attempted to connect to Google News through an API key but it was far too difficult to extract the news article content so I instead switched to the PyGoogleNews library.

3D GRAPHICS MODELLING
Project Link
Tech Stack
Python:
Graphics
Overview

Model object and material files as 3D graphical components, and manipulate them with a user interface that allows for object transformations.

Utilise vector mathematics to render lighting and implement shading depth to objects.

Process

  1. Load object and material files and render them in a user interface.
  2. Implement rendering pipelines for vertex buffers and shaders.
  3. Manipulate transformations like rotate, scale and translate to interact with 3D objects through the user interface.
  4. Apply vector mathematics for light calculations that were then implemented with shaders.

Challenges

  1. Understanding how to render 3D objects by linking object files to their respective material files.
  2. Customising shaders to render lighting effects for 3D objects, depending on the vector mathematics calculated.

DATA ANALYTICS WEBAPP
Project Link
Tech Stack
Python:
Web Devlopment
Backend and Data Analysis
Overview

Build a webapp that allowed music managers to manage their artists and touring schedules more effectively.

There was a focus data management and optimisation, achieved through buidling a pipeline of uploading tour information and then providing data analytical insights for data-driven decision making.

Deployed at: https://hothack.onrender.com/users/login

Process

  1. Create an analytics dashboard interface with modern 'card' designs.
  2. Link the UI to the backend database to query data from SQLite.
  3. Add interactivity and functionality to the dashboard such as filtering the data displayed.

Challenges

  1. The Ruby on Rails framework inherently implements the MVP paradigm so it took some time to understand how the syntax allows for this communication between the frontend and the backend.
  2. It took awhile to be able to ensure consistent dependencies were used across each developer's local version as we were quickly iterating through the project where there were constantly GitHub pull requests and merge conflicts.
  3. Implementing interactivity for the analytics interface was quite difficult as it required continuous queries to the database and then re-rendering the frontend.
